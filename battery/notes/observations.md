# battery 観察ログ

`_bat_debug` の実測を時系列で append する。過去エントリは編集しない
(時系列の信頼性を保つため)。テンプレは `../README.md` 参照。

各行の読み方:
`V` 端子電圧(VDC) / `I` 電流(A) / `state` charging|discharging|full /
`ocv` 内部抵抗補正後の開放電圧推定(mV) / `soc(factory)` v3 OCV table 補間 SOC /
`soc(raw)` kernel `percent0` そのまま。

---

## 2026-05-30 02:58 JST  (v3-factory-R045 適用後、 残量帯 ~98-99%)

- 充電中 (0.55A): `V=4.20 I=0.55 state=charging R=0.045 ocv=4175 soc(factory)=98% soc(raw)=99%`
  - PS1 表示: `[+98%]` / raw: `99%`
  - 生センサ: `volt0=4.20 VDC` / `current0=0.55 A (charging)` / `raw0=3 (charging)` / `percent0=99.00%`
- 環境: 室温 ~25°C、 uptime 48 分、 ユーザ 1 (sshd 経由)、 充電器は接続中
  (種類は未記録 — 次回 USB / 純正を明記する)
- 結論: 満充電付近では factory(98%) と raw(99%) が 1% 差でほぼ一致。
  端 (>95%) でも v2 までの乖離は見られず。 **充電中 1 点のみの baseline 計測**で、
  plug/unplug の jump 幅は未計測。

### この計測の位置づけ / 残タスク
- これは v3 snapshot を `kshrc/v3-factory-R045.kshrc` として確定した直後の
  baseline。 snapshot は実機正本と sha256 一致を確認済。
- **未計測**: 放電中 (-I) の値、 および plug → unplug 直後の SOC jump 幅。
  README「PS1 `_bat` の進化」表の v3 精度 (±10-20%) を裏取りするには、
  - 放電状態 (充電器を抜いて負荷をかけた状態) の `_bat_debug`
  - 抜き差し直後 10s の SOC 変化
  を別セッションで取る必要がある。 実機操作 (充電器の抜き差し) を伴うため、
  ここでは充電中の 1 点に留めた。

## 2026-05-30 03:33 JST  (v3-factory-R045 適用後、 残量帯 ~80-84%)

- 放電中 (0.45A): `V=4.04 I=0.45 state=discharging R=0.045 ocv=4060 soc(factory)=84% soc(raw)=80%`
  - 生センサ: `volt0=4.04 VDC` / `current0=0.45 A (discharging)` / `raw0=5 (discharging)` / `percent0=80.00%`
- 環境: 室温 ~25°C、 02:58 の充電セッションから充電器を抜いた後の自然放電
  (負荷 = sshd + Wi-Fi のみ、 0.45A)
- 結論: **放電時は factory(84%) > raw(80%)** で +4% 差。 02:58 充電時は
  factory(98%) < raw(99%) で −1% 差だったので、 **OCV 補正の符号は正しい向き**
  (放電で sag した端子電圧を `V + I·R` で上方補正、 充電で浮いた電圧を下方補正)。
  R=0.045 / 工場 21 点 table が両極性で機能していることを確認。

### この計測で言えること / まだ言えないこと
- **言える**: 放電・充電の両状態で OCV 補正が正しい向きに効く。 raw との差は
  この残量帯で ±1〜4%。 v3 は v1 (raw 直読み、 ±33% jump) より明確に安定。
- **まだ言えない (純 jump 幅)**: 02:58→03:33 は 35 分空いており、 98→84% の
  低下は「plug/unplug の瞬間ジャンプ」と「実放電」が混ざっている。 純粋な
  jump 幅 (README の ±10-20% 主張の核) を取るには、 **同一時刻に充電器を抜き
  差しして 10 秒以内の factory/raw 変化**を観測する必要がある。 これは次の
  計測セッションのタスクとして残す。 → **下の 03:35-03:37 で計測済**。

## 2026-05-30 03:35-03:37 JST  (v3-factory-R045、 残量帯 ~82-85%) — plug/unplug 純 jump 計測

同一残量帯 (真 SOC ~83-84% でほぼ不変) で充電器を抜き差しし、 10〜15 秒以内の
即時変化を観測。 真値が動かない時間窓なので、 差分はそのまま「状態遷移による
表示ジャンプ」を表す。

| 時刻 | 状態 | V | I | ocv | factory | raw |
|---|---|---:|---:|---:|---:|---:|
| 03:35:33 | 放電 (基準) | 4.02 | 0.59 | 4046 | 82% | 78% |
| 03:36:0x (+0〜15s) | 充電 (挿入直後、 安定) | 4.19 | 0.80-0.82 | 4153 | 96% | 99% |
| 03:37:51 (+0〜15s) | 放電 (抜去直後、 収束) | 4.04 | 0.45 | 4060 | 84% | 81% |

- **挿入 jump**: raw `78→99` = **+21pt**、 factory `82→96` = **+14pt**
- **抜去 jump**: raw `99→81` = **−18pt**、 factory `96→85` = **−11pt**
- **振幅**: 真 SOC ~83% 一定にもかかわらず、 raw は 78〜99 (振幅 21pt)、
  factory は 82〜96 (振幅 14pt) を往復。
- 結論:
  - **v3 OCV 補正は jump を raw の ~21pt → ~14pt に圧縮**するが **消さない**。
    この 80-95% 帯では `V − I·R` (R=45mΩ) が充電分極 (CV 寄り) を取り切れず、
    補正不足。 README の「v3 = ±10-20%、 端 (>95%) で誤差大」と実測一致。
  - 放電に戻ると factory は ~84-85% に収束し挿入前 82% とほぼ一致 = **factory は
    充電ノイズの影響が小さく真値追従性が高い**。 raw は端子電圧そのままなので
    充電中 99% に張り付く。
  - これが **v4 (HW クーロンカウンタ駆動) でないと根治しない**ことの
    実測根拠。 端子電圧ベースの OCV 補正の限界 = ±10-15pt の plug jump。
- 環境: 室温 ~25°C、 負荷 sshd + Wi-Fi (放電 0.45-0.59A)、 充電器は USB
  (電流 0.8A 級、 純正/USB の別は未記録)。

## 2026-05-30 03:41 JST  (v3-factory-R045、 残量帯 ~83%) — 内部抵抗 R キャリブレーション

充電器を抜いた純放電状態で **負荷ステップ法** (CPU を `dd` 4 本で焼いて放電
電流を上げ、 同一 SOC で `R = ΔV / ΔI`) を実施。 真 SOC は ~82-83% で不変
(load 解除後 4.03V/83% に復帰し確認)。

| 状態 | V | I (放電) | factory | raw |
|---|---:|---:|---:|---:|
| idle (3 点平均) | 4.03 | 0.48 | 83% | 79% |
| load (dd×4, 3 点平均) | 4.01 | 0.68 | 81% | 76% |
| 解除後 settle | 4.03 | 0.45 | 82% | 79% |

- **負荷ステップ法**: ΔV = 4.03−4.01 = 0.023V、 ΔI = 0.68−0.48 = 0.194A
  → **R ≈ 0.119 Ω (119 mΩ)**
- **充電/放電 2 点法** (03:35-37 jump 実験データ、 同一 ~83% SOC):
  R = (V充−V放)/(I充+I放) = (4.19−4.02)/(0.82+0.59) = 0.17/1.41
  → **R ≈ 0.121 Ω (121 mΩ)**
- → **2 手法が一致して R ≈ 0.12 Ω**。 工場 dtb `bat_res = 0x2d` (45 mΩ) の
  約 2.6 倍。 工場値は新品セルの瞬時オーミック、 実測は経年+分極込みの実効抵抗。

### 含意 — `_BAT_R` を 0.12 に上げると jump がほぼ消える
03:35-37 の plug 点で R=0.12 を当てた検算:
- 充電点 ocv = 4.19 − 0.82×0.12 = **4.092V** → ~88%
- 放電点 ocv = 4.02 + 0.59×0.12 = **4.091V** → ~88%
- 両方ほぼ一致 → **plug jump ≈ 0** (R=45mΩ では 96% vs 82% = 14pt 残っていた)。

この R は `_bat` が使うのと同一の電流センサ (`current0`) で較正しているため、
電流 ADC の絶対精度がずれていても OCV 補正は **自己整合的**に効く。

## 2026-05-30 03:57-03:58 JST  (v3-cal-R120 デプロイ後、 残量帯 ~86%) — jump 再計測で較正を実証

`kshrc/v3-cal-R120.kshrc` (R=0.12) を実機投入後、 03:35-37 と同じ plug 計測を
同一残量帯 (真 SOC ~86% 不変) で再実施。

| 時刻 | 状態 | V | I | ocv | factory | raw |
|---|---|---:|---:|---:|---:|---:|
| 03:57:30 | 放電 (基準) | 4.01 | 0.52 | 4072 | 86% | 77% |
| 03:58:0x (+0〜15s) | 充電 (挿入直後、 安定) | 4.19 | 0.95-0.99 | 4071-4076 | 85-86% | 99% |

- **挿入 jump**: raw `77→99` = **+22pt** (R を使わないので不変)、
  factory `86→85-86` = **~0〜1pt**
- OCV 推定が放電 4072mV / 充電 4071-4076mV で**ほぼ同一** → factory はフラット。
- **R=0.045 (v3) との直接比較** (同じ ~83-86% 帯、 同じ操作):

  | | R=0.045 (03:35-37) | R=0.12 (03:57-58) |
  |---|---:|---:|
  | factory plug jump | +14pt (82→96) | **~0-1pt (86→86)** |
  | raw plug jump | +21pt | +22pt (不変) |

- **結論: 較正成功**。 実測 R=0.12 で plug jump (端子電圧ベース補正の最大の
  弱点) が 14pt → ~0pt に消失。 raw は依然 22pt 跳ねるので、 改善は純粋に
  OCV 補正 (R) によるもの。 これで v3-cal の plug jump 精度は **実測で ~±1pt**
  (「期待」ではなく確定値)。
- 残る誤差源: OCV table 自体の精度、 SOC・温度による R 変動、 電圧 10mV 分解能。
  端 (>95% / <10%) は未計測。 ただし plug jump という最大要因は除去できた。
- 環境: 室温 ~25°C、 放電 0.52A (sshd+Wi-Fi)、 充電 ~0.95-0.99A (USB)。

### 注意 / 限界
- 電圧分解能が 10mV と粗く、 ΔV~20mV の測定なので R の不確かさは ±20% 程度
  (~100-150 mΩ)。 2 手法一致と jump 検算で「45mΩ より 0.12Ω が明確に良い」
  ことは確実だが、 0.12 という値自体は ~83% SOC・室温・~8s パルスの 1 点。
- SOC・温度で R は変わる。 厳密には残量帯ごとに測るべきだが、 v3 (端子電圧
  ベース) の延命策としては単一定数 0.12 で十分な改善。 根治は HW クーロン
  カウンタ駆動 (kernel-patches/ の RK818 driver patch 系)。
